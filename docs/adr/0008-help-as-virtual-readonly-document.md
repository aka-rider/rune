# Help is a virtual read-only document, generated from the keymap

Help is presented as a **read-only document opened as a tab in the main editor**, not as an overlay, a modal, or a top-level router page. It is opened and toggled with `^?` (fallback `F1`), reached with `^1..9`, and closed with `^w` — identical to any file. It is virtual: no file on disk, and it is excluded from the journal/snapshot/autosave model, so it never goes dirty and never trips the quit path.

Its content is the **single source of truth, the keymap**. The keybindings reference is generated by reflecting over every `key.Binding` field of `keymap.Bindings` (`.Help().Key` + `.Help().Desc`), in struct-declaration order. The only authored material is prose that cannot be derived from the keymap (intro, voice input, Obsidian vaults), embedded as a `//go:embed` markdown template with one splice point for the generated list. The same reflection walk re-backs `AllPhysicalKeys()` and the collision validator, and the footer's hint line renders from binding glyphs — retiring every hand-maintained key list.

This supersedes the previous help mechanism: the `?`-bound `HelpExpand` footer toggle, the `helpExpanded`/`helpEntries` rendering, `keymap.HelpText()`, and the keybinding tables in `README.md` (replaced by a pointer to in-app help).

Why a tab and not an overlay/page: it reuses the read-only `markdownedit` path already proven by chat (scroll, `⌘F` search, copy for free), it inherits tabs/`^w`/`^1..9` muscle memory, and `app.go` is a thin single-page wrapper today — a router page would mean building routing infrastructure for one screen. An overlay would be a parallel rendering surface to maintain.

Why reflection and not a curated list: the original bug and the README drift both trace to parallel sources of truth (a binding lived in one place, its documentation in another, and they diverged). A curated `HelpSections()` or a template with per-key descriptions would reintroduce exactly that — a new binding would inevitably be forgotten. Reflection makes omission structurally impossible. Grouping is dropped for now; if wanted later it must stay single-source (e.g. a `help:"group=…"` struct tag beside the field), never a second list.

Consequence: the main editor is overloaded — it shows either the active file *or* the help document, while the journal's `main` surface still refers to the active file. This requires three guards in the workspace: (1) the help sentinel path loads generated content from memory instead of `os.ReadFile` at the `requestOpenPath`/`loadFileCmd` chokepoint; (2) undo/redo no-op while the help tab is active, so `^Z` cannot target the help buffer; (3) `EnsureDocument`, title-rename, and chat file-context are skipped for the sentinel path. `ctrl+?` must be verified per-terminal (caret `^?` is classically DEL/Backspace); `F1` is the guaranteed-deliverable fallback.
